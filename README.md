# WinFormsKursach — каталог мобильных телефонов

Приложение Windows Forms: каталог телефонов с фильтрами и сортировкой, карточка товара и оформление заказа. Данные загружаются из базы Microsoft Access с тремя таблицами: **Phones** (id, Model), **Characteristics** (id_phone, Producer, Os, ScreenSize, RamGb, …), **Shops** (id_phone, Shop, Price).

---

## База Access (схема данных)

- **Phones** — id (счётчик, PK), Model (название модели). Связь 1 ко многим с Characteristics и Shops.
- **Characteristics** — id (PK), id_phone (FK → Phones), Producer, Os, ScreenSize, RamGb, StorageGb, ImageUrl, InStock, ScreenTech, ScreenResolution, ScreenRefreshHz, Processor, CameraMp, BatteryMah, Waterproof, Description.
- **Shops** — id (PK), id_phone (FK → Phones), Shop (название магазина), Price (денежный).

---

## Структура проекта

| Компонент | Назначение |
|-----------|------------|
| **Form1** | Каталог: таблица телефонов, фильтры, сортировка, поиск, переход в карточку |
| **Form2** | Карточка товара: характеристики, список магазинов и цен из Shops, кнопка «Купить» → Form3 |
| **Form3** | Оформление заказа: данные заказа, ФИО, телефон, кнопка «Купить» |
| **Helpers.DbHelper** | Путь к базе и строка подключения OleDb |
| **Helpers.PhoneImageHelper** | Общая загрузка изображения телефона в PictureBox |
| **Models.Phones** | Модель данных одного телефона (заполняется из Phones + Characteristics + цена из Shops) |
| **Program** | Точка входа: запуск Form1 |

## Form1 — каталог (главная форма)

Хранит полный список телефонов в поле `_allPhones`, загружает данные из Access, отображает отфильтрованный/отсортированный список в `DataGridView` через `BindingSource`.

### Конструктор `Form1()`

**Сигнатура:** `public Form1()`

**Назначение:** Инициализация формы каталога и привязка обработчиков событий.

**Действия:**
- Вызов `InitializeComponent()` (создание контролов из Designer).
- Двойной щелчок по таблице и кнопка «Подробнее / Купить» открывают карточку: `dgvPhones.DoubleClick`, `button1.Click` → `OpenDetail()`.
- Кнопка «Сбросить фильтры» → `resetFilters_Click`.
- Смена сортировки → `cbSort_SelectedIndexChanged` → `ApplyFiltersAndSort()`.
- Любое изменение фильтров (производитель, ОС, цена, «только в наличии», поиск) → `FiltersChanged` → `ApplyFiltersAndSort()`.

**Когда вызывается:** При запуске приложения (создание главной формы в `Program.Main`).

### `LoadPhonesFromDB()`

**Сигнатура:** `public void LoadPhonesFromDB()`

**Назначение:** Загрузка каталога из Access: объединение таблиц Phones и Characteristics, цена — минимум по таблице Shops. Результат сохраняется в `_allPhones`.

**Алгоритм:**
1. Проверка наличия файла базы через `DbHelper.GetDbPath()`. Если файла нет — выход.
2. Запрос: `FROM Phones P INNER JOIN Characteristics C ON C.id_phone = P.id`, в выборке — P.id, P.Model, все поля C; подзапрос `(SELECT MIN(S.Price) FROM Shops S WHERE S.id_phone = P.id) AS Price`.
3. Для каждой строки создаётся объект `Phones` (Id, Name=Model, поля из C, Price из подзапроса). Для опциональных полей используются `GetString` и `GetInt`.
4. Список присваивается полю `_allPhones`.

**Когда вызывается:** Из обработчика загрузки формы `Form1_Load`.

**Зависимости:** `DbHelper.GetDbPath()`, `DbHelper.GetConnectionString()`, методы `GetString` и `GetInt`.

---

### `GetString(OleDbDataReader reader, string col)`

**Сигнатура:** `private static string GetString(OleDbDataReader reader, string col)`

**Назначение:** Безопасное чтение строкового значения колонки из DataReader.

**Параметры:**
- `reader` — открытый DataReader после `ExecuteReader()`.
- `col` — имя колонки (например, `"ScreenTech"`, `"Description"`).

**Возвращает:** Строку без пробелов по краям (`Trim`), либо пустую строку при `DBNull`, отсутствии колонки или исключении.

**Когда вызывается:** В цикле чтения в `LoadPhonesFromDB()` для полей ScreenTech, ScreenResolution, Processor, Waterproof, Description.

---

### `GetInt(OleDbDataReader reader, string col)`

**Сигнатура:** `private static int GetInt(OleDbDataReader reader, string col)`

**Назначение:** Безопасное чтение целочисленного значения колонки из DataReader.

**Параметры:** Аналогично `GetString`.

**Возвращает:** Число типа `int` или 0 при `DBNull`, отсутствии колонки или ошибке.

**Когда вызывается:** В `LoadPhonesFromDB()` для ScreenRefreshHz, CameraMp, BatteryMah.

---

### `FillPhonesList(IEnumerable<Phones> phones)`

**Сигнатура:** `private void FillPhonesList(IEnumerable<Phones> phones)`

**Назначение:** Обновление отображения таблицы: привязка переданного списка телефонов к `DataGridView` через `phonesBindingSource`.

**Параметры:** `phones` — отфильтрованный и/или отсортированный список (или полный каталог).

**Действия:** `phonesBindingSource.DataSource = phones.ToList();` — таблица на форме показывает этот список.

**Когда вызывается:** Из `ApplyFiltersAndSort()` после построения запроса по фильтрам и сортировке.

---

### `Form1_Load(object sender, EventArgs e)`

**Сигнатура:** `private void Form1_Load(object sender, EventArgs e)`

**Назначение:** Обработчик события загрузки формы: первичная загрузка данных из Access, настройка фильтров и отображение списка.

**Действия:**
1. `LoadPhonesFromDB()` — заполнение `_allPhones` из БД.
2. `FillFilterCombos()` — заполнение комбобоксов «Сортировка», «Производитель», «ОС» и границ цены.
3. `ApplyFiltersAndSort()` — применение фильтров (в т.ч. по умолчанию) и вывод результата в таблицу.

**Когда вызывается:** Автоматически при первом отображении Form1 (событие Load).

---

### `FillFilterCombos()`

**Сигнатура:** `private void FillFilterCombos()`

**Назначение:** Заполнение выпадающих списков и диапазона цены уникальными значениями из текущего каталога `_allPhones`.

**Действия:**
- **Сортировка:** пункты «По цене (сначала дешёвые)», «По цене (сначала дорогие)», «Модель А-Я», «Модель Я-А»; выбран первый.
- **Производитель:** «Все» + уникальные значения `Producer`, отсортированные по алфавиту.
- **ОС:** «Все» + уникальные значения `Os`, отсортированные.
- **Цена:** если в каталоге есть телефоны — для мин. цены задаются `Minimum = 0`, `Maximum` как у макс. цены, `Value = 0`; для макс. цены — `Maximum = max(макс. цена * 2, 100000)`, `Value = макс. цена`. Так фильтр минимальной цены доступен во всём диапазоне.

**Когда вызывается:** Один раз при загрузке формы из `Form1_Load`. Предполагается, что после загрузки каталог не меняется, поэтому повторный вызов при смене данных не делается.

---

### `ApplyFiltersAndSort()`

**Сигнатура:** `private void ApplyFiltersAndSort()`

**Назначение:** Применение всех фильтров и выбранной сортировки к `_allPhones` и обновление таблицы.

**Алгоритм:**
1. Исходная последовательность — все телефоны из `_allPhones`.
2. Если в «Производитель» выбрано не «Все» — отбор по равенству `Producer`.
3. Если в «ОС» выбрано не «Все» — отбор по равенству `Os`.
4. Отбор по цене: `Price` между `numPriceMin.Value` и `numPriceMax.Value`.
5. Если включено «Только в наличии» — только записи с `InStock == true`.
6. Если строка поиска не пуста — отбор по вхождению подстроки в название или производителя (без учёта регистра).
7. Сортировка по индексу комбобокса: 0 — по цене по возрастанию, 1 — по цене по убыванию, 2 — по названию А–Я, 3 — по названию Я–А.
8. Результат передаётся в `FillPhonesList(query.ToList())`.

**Когда вызывается:** Из `Form1_Load`, из `resetFilters_Click`, при смене сортировки (`cbSort_SelectedIndexChanged`) и при любом изменении фильтров (`FiltersChanged`).

---

### `resetFilters_Click(object? sender, EventArgs e)`

**Сигнатура:** `private void resetFilters_Click(object? sender, EventArgs e)`

**Назначение:** Сброс всех фильтров к значениям по умолчанию и перестроение списка.

**Действия:**
- Производитель и ОС — «Все» (индекс 0).
- Минимальная цена — `numPriceMin.Minimum`, максимальная — снова максимум по каталогу (если он не пуст).
- Снята галочка «Только в наличии», очищено поле поиска.
- Вызов `ApplyFiltersAndSort()` для обновления таблицы.

**Когда вызывается:** По нажатию кнопки «Сбросить фильтры».

---

### `cbSort_SelectedIndexChanged(object? sender, EventArgs e)`

**Сигнатура:** `private void cbSort_SelectedIndexChanged(object? sender, EventArgs e) => ApplyFiltersAndSort();`

**Назначение:** Обработчик смены пункта сортировки — перестраивает список с учётом текущих фильтров и новой сортировки.

**Когда вызывается:** При изменении выбора в комбобоксе «Сортировка».

---

### `FiltersChanged(object? sender, EventArgs e)`

**Сигнатура:** `private void FiltersChanged(object? sender, EventArgs e) => ApplyFiltersAndSort();`

**Назначение:** Общий обработчик изменения любого из фильтров (производитель, ОС, границы цены, «только в наличии», текст поиска) — перестраивает список.

**Когда вызывается:** Из событий комбобоксов производителя и ОС, числовых полей цены, чекбокса и поля поиска (подписаны в конструкторе).

---

### `OpenDetail()`

**Сигнатура:** `private void OpenDetail()`

**Назначение:** Открытие карточки товара (Form2) для выбранной в таблице строки.

**Алгоритм:**
1. Если в таблице ничего не выбрано (`dgvPhones.CurrentRow == null`) — показ сообщения «Выберите телефон из списка» и выход.
2. Из текущей строки берётся объект: `(Phones)dgvPhones.CurrentRow.DataBoundItem`.
3. Создаётся и показывается модально форма `Form2(phone)` через `ShowDialog()`.

**Когда вызывается:** По двойному щелчку по строке таблицы или по кнопке «Подробнее / Купить».

## Form2 — карточка товара

Показывает характеристики выбранного телефона, список магазинов и цен из таблицы **Shops** (по id телефона) и кнопку перехода к оформлению заказа (Form3). Данные о телефоне берутся из переданного при открытии объекта `Phones`.

### Конструктор `Form2()`

**Сигнатура:** `public Form2()`

**Назначение:** Инициализация формы карточки и привязка событий.

**Действия:**
- `InitializeComponent()`.
- При загрузке формы — `Form2_Load`.
- При смене выбранной строки в списке магазинов — `listView2_SelectedIndexChanged` → обновление текста кнопки «Купить за».
- Кнопка «Назад» — закрытие формы.
- Кнопки (например, «Подробнее» и «Купить за») — открытие формы заказа: `OpenOrderForm`.

**Когда вызывается:** Не вызывается напрямую пользовательским кодом; используется конструктор `Form2(Phones phone)`.

---

### Конструктор `Form2(Phones phone)`

**Сигнатура:** `public Form2(Phones phone) : this()`

**Назначение:** Создание карточки для конкретного телефона.

**Параметры:** `phone` — объект с данными телефона (название, характеристики, цена и т.д.).

**Действия:** Вызывает параметрный конструктор `this()`, затем сохраняет переданный объект в поле `_phone`. Заполнение контролов происходит при загрузке формы в `Form2_Load`.

**Когда вызывается:** Из Form1 в `OpenDetail()` после выбора строки в каталоге.

---

### `Form2_Load(object? sender, EventArgs e)`

**Сигнатура:** `private void Form2_Load(object? sender, EventArgs e)`

**Назначение:** Заполнение карточки при первом отображении: заголовок, характеристики, список магазинов с ценами, изображение, текст кнопки «Купить за».

**Действия (если `_phone != null`):**
1. Заголовок: `label1.Text = _phone.Name`; кнопка по умолчанию: «Купить за {цена} Р».
2. Очистка и заполнение таблицы характеристик (`listView1`) через `AddChar`: название, производитель, ОС, экран, тип матрицы, разрешение, ОЗУ, память, процессор, камера, батарея, влагозащита; при непустом описании — строка «Описание».
3. Очистка списка магазинов и загрузка из БД: `LoadShopsForPhone(_phone.Id)` заполняет `listView2` из таблицы Shops (Shop, Price); выбор первой строки при наличии.
4. Загрузка изображения: `PhoneImageHelper.Load(pictureBox1, _phone)`.
5. Обновление текста кнопки «Купить за» по выбранному магазину: `UpdateBuyButtonText()`.

**Когда вызывается:** Один раз при показе формы (событие Load).

---

### `listView2_SelectedIndexChanged(object? sender, EventArgs e)`

**Сигнатура:** `private void listView2_SelectedIndexChanged(object? sender, EventArgs e)`

**Назначение:** Обработчик смены выбранного магазина в списке — обновляет текст кнопки «Купить за» под цену выбранного магазина.

**Действия:** Вызов `UpdateBuyButtonText()`.

**Когда вызывается:** При выборе другой строки в списке магазинов (listView2).

---

### `UpdateBuyButtonText()`

**Сигнатура:** `private void UpdateBuyButtonText()`

**Назначение:** Установка текста кнопки «Купить за» в зависимости от выбранной строки в списке магазинов.

**Действия:**
- Если ни одна строка не выбрана — текст «Купить».
- Иначе берётся цена из второй колонки выбранной строки (SubItems[1]) и подставляется в строку вида «Купить за {цена} Р».

**Когда вызывается:** Из `Form2_Load` и из `listView2_SelectedIndexChanged`.

---

### `LoadShopsForPhone(int phoneId)`

**Сигнатура:** `private void LoadShopsForPhone(int phoneId)`

**Назначение:** Загрузка из таблицы Shops магазинов и цен для телефона с указанным id и заполнение listView2 (колонки: магазин, цена).

**Параметры:** `phoneId` — id телефона в таблицах Phones/Shops.

**Действия:** Подключение к базе через `DbHelper`, запрос `SELECT Shop, Price FROM Shops WHERE id_phone = ?`, добавление строк в listView2. Если файла базы нет — выход без изменений.

**Когда вызывается:** Из `Form2_Load` после заполнения характеристик.

---

### `AddChar(string param, string value)`

**Сигнатура:** `private void AddChar(string param, string value)`

**Назначение:** Добавление одной строки «параметр — значение» в таблицу характеристик (listView1).

**Параметры:**
- `param` — название параметра (например, «ОЗУ», «Разрешение»).
- `value` — значение (при `null` подставляется пустая строка).

**Действия:** В listView1 добавляется новая строка с двумя колонками: param и value.

**Когда вызывается:** Многократно из `Form2_Load` при заполнении блока характеристик.

---

### `OpenOrderForm(object? sender, EventArgs e)`

**Сигнатура:** `private void OpenOrderForm(object? sender, EventArgs e)`

**Назначение:** Открытие формы оформления заказа (Form3) с выбранным магазином и ценой.

**Алгоритм:**
1. Если `_phone == null` — выход.
2. Если в списке магазинов ничего не выбрано — сообщение «Выберите магазин для покупки» и выход.
3. Из выбранной строки берётся название магазина (первая колонка) и цена (вторая колонка); при неудачном разборе числа подставляется `_phone.Price`.
4. Создаётся и показывается модально `Form3(_phone, storeName, price)`.

**Когда вызывается:** По нажатию кнопок, ведущих в заказ (например, «Купить за …»).

## Form3 — оформление заказа

Показывает сводку заказа (телефон, магазин, цена, количество, итого), поля ФИО и телефона, кнопку «Купить» и «Назад». После успешной проверки данных показывает сообщение об оформлении и закрывается.

**Действия:**

- Кнопка «Купить» — `form3btnBuy_Click`.
- Кнопка «Назад» — закрытие формы.

**Когда вызывается:** Из других конструкторов Form3 через `: this()`.

---

**Сигнатура:** `public Form3(Phones phone, string storeName, decimal price) : this()`

**Назначение:** Создание формы заказа для выбранного телефона, магазина и цены.

**Параметры:**
- `phone` — товар.
- `storeName` — название магазина (при `null` подставляется «Каталог»).
- `price` — цена за единицу.

**Действия:** Сохраняет аргументы в поля `_phone`, `_storeName`, `_unitPrice`. Заполнение интерфейса — в `Form3_Load`.

**Когда вызывается:** Из Form2 в `OpenOrderForm` после выбора магазина.

---

### `Form3_Load(object? sender, EventArgs e)`

**Сигнатура:** `private void Form3_Load(object? sender, EventArgs e)`

**Назначение:** Заполнение блока «Ваш заказ» при первом отображении формы.

**Действия (если `_phone != null`):**

3. Загрузка изображения: `PhoneImageHelper.Load(pictureBox1, _phone)`.

**Когда вызывается:** Один раз при показе Form3 (событие Load).

---

### `UpdateTotal()`

**Сигнатура:** `private void UpdateTotal()`

**Назначение:** Обновление подписи количества и итоговой суммы по текущему значению NumericUpDown.

**Действия:** Читается `(int)numericUpDown1.Value`; в label9 выводится «Количество: {qty} шт.»; в label10 — «Итого: {_unitPrice * qty} Р» с форматом числа.

**Когда вызывается:** Из `Form3_Load` и при каждом изменении количества (ValueChanged).

### `form3btnBuy_Click(object? sender, EventArgs e)`

**Сигнатура:** `private void form3btnBuy_Click(object? sender, EventArgs e)`

**Назначение:** Проверка введённых ФИО и номера телефона; при успехе — сообщение «Заказ оформлен» и закрытие формы.

**Валидация:**
1. ФИО: поле не должно быть пустым или состоять только из пробелов; иначе — предупреждение «Введите ФИО» и выход.
2. Телефон: из текста поля считаются только цифры; их должно быть не менее 9. Пустое поле, только плейсхолдер «+375( )» или меньше 9 цифр — предупреждение «Введите номер телефона (не менее 9 цифр)» и выход.

**При успехе:** MessageBox «Заказ оформлен», затем `Close()`.

**Когда вызывается:** По нажатию кнопки «Купить» на форме заказа.

## Helpers — общие вспомогательные методы

### `DbHelper.GetDbPath()` и `DbHelper.GetConnectionString()`

**Назначение:** Единая точка доступа к базе Access. `GetDbPath()` возвращает путь к файлу `Data\PhonesDB1.accdb` рядом с exe. `GetConnectionString()` возвращает строку подключения OleDb для этого файла.

**Когда вызывается:** Из Form1 в `LoadPhonesFromDB`, из Form2 в `LoadShopsForPhone`.

---

### `PhoneImageHelper.Load(PictureBox box, Phones phone)`

**Сигнатура:** `public static void Load(PictureBox box, Phones phone)`

**Назначение:** Загрузка изображения телефона из папок Data и Data\Images в переданный PictureBox.

**Параметры:**
- `box` — контрол, в который выводится изображение.
- `phone` — объект телефона (используются `Id` и `ImageUrl`).

**Алгоритм:**
1. Определяются пути: `Application.StartupPath\Data` и `…\Data\Images`.
2. Имя файла: из `phone.ImageUrl` (если не пусто), иначе `{phone.Id}.jpg`; из имени берётся базовое имя без расширения.
3. Перебираются варианты: `baseName.jpg` и `baseName.png` в Data и в Data\Images (в указанном порядке).
4. Для первого существующего файла изображение загружается, клонируется в `box.Image` (чтобы не держать файл открытым), цикл прерывается. При ошибке чтения файла переходим к следующему пути.

**Когда вызывается:** Из Form2 в `Form2_Load` и из Form3 в `Form3_Load`.

## Модель данных Phones

Класс в пространстве имён `WinFormsKursach.Models` без методов; только свойства для хранения данных одного телефона. Имена свойств совпадают с колонками в Access (кроме названия модели: в БД колонка **Model**, в классе — **Name**, т.к. «Name» в Access зарезервировано).

| Свойство | Тип | Описание |
|----------|-----|----------|
| Id | int | Идентификатор |
| Name | string | Название модели (в БД — колонка Model) |
| Producer | string | Производитель |
| Price | decimal | Цена |
| Os | string | Операционная система |
| ScreenSize | string | Размер/диагональ экрана |
| ScreenTech | string | Тип матрицы (AMOLED, IPS и т.д.) |
| ScreenResolution | string | Разрешение экрана |
| ScreenRefreshHz | int | Частота обновления, Гц |
| Processor | string | Процессор |
| RamGb | int | ОЗУ, ГБ |
| StorageGb | int | Встроенная память, ГБ |
| CameraMp | int | Камера, Мп |
| BatteryMah | int | Ёмкость батареи, мА·ч |
| Waterproof | string | Влагозащита (IP68 и т.д.) |
| ImageUrl | string | Имя файла/URL изображения |
| Description | string | Описание |
| InStock | bool | В наличии |

## Поток данных

1. **Старт** — `Program.Main()` → открывается Form1.
2. **Form1_Load** → `LoadPhonesFromDB()` заполняет `_allPhones` из Access → `FillFilterCombos()` → `ApplyFiltersAndSort()` выводит список в таблицу.
3. Пользователь меняет фильтры/сортировку → `ApplyFiltersAndSort()` → `FillPhonesList()` обновляет таблицу.
4. Выбор строки и двойной щелчок или «Подробнее / Купить» → `OpenDetail()` → Form2(phone).
5. **Form2_Load** заполняет карточку и список магазинов; пользователь выбирает магазин и нажимает «Купить за» → `OpenOrderForm()` → Form3(phone, storeName, price).
6. **Form3_Load** заполняет блок заказа; пользователь вводит ФИО и телефон, нажимает «Купить» → `form3btnBuy_Click()` проверяет данные → при успехе сообщение и закрытие Form3 (при необходимости затем Form2).

Изображения везде подгружаются через `PhoneImageHelper.Load` из папок Data и Data\Images по `Id` или `ImageUrl`.
